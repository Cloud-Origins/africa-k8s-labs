# kubectl Tooling, Contexts & Dashboard

Track: KCNA, CKAD  |  Estimated time: 1.5h  |  Primary tool: kubectl  |  Type: Lab

Grading: manual. There is no autograder for this lab; your mentor reviews your show-off checkpoint in your PLG space.

## Objective

Work faster with kubectl: contexts, default namespaces, output formats, explain, and plugins.

## Your task

Set your context default namespace; extract a single field with -o jsonpath; use 'kubectl explain' to inspect a resource schema; install one krew plugin (for example 'kubectl krew install ns'); and optionally enable the Kubernetes Dashboard.

## Show-off checkpoint

Share a terminal clip showing a jsonpath query you wrote and one krew plugin you installed in use.
